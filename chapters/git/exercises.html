18.8. Exercises: Git More Practice¶

18.8.1. Working with a Local Repository¶

Use your terminal powers to move through these Git exercises.

  1. Launch Visual Studio Code and open the terminal pane.

  2. In your local_practice folder, make a new directory called Git-Exercises. Next, go to File –> Open and select this new folder.

  3. In the terminal, make sure that you are in the Git-Exercises directory. Initialize it as a new repository with git init. If you wish, change the name of the default branch to main.

  4. Add two files to the directory: index.html and style.css. Use the status/add/commit commands to commit the changes to the repository.

  5. Paste in this boilerplate HTML code into index.html, then commit again.

  6. Use git checkout -b to create a new branch called html-code. While in that branch, add at least 5 new elements to the HTML file. Save and commit your changes.

  7. Return to main. Use checkout again to create a third branch called css-styles. While in this new branch, open the CSS file and add style rules for at least three different elements. Also, include at least one class selector. Save and commit your changes.

You should have 3 branches in your repository now:

$ git branch
* css-styles
  html-code
  main

18.8.1.1. Merging¶

Notice that none of the three branches contains a complete set of code. main and html-code have a blank style.css file, while main and css-styles do not have the updated HTML. You will fix this with two merges.

  1. Return to main and use the git merge command pull in the changes from html-code.
  2. Check inside index.html and use git log in the terminal to confirm that main contains the commits you made in the other branch.
  3. Repeat steps 1 and 2 for the css-styles branch.
  4. Can you merge the same two branches again? Find out by moving into the html-code branch and adding more code and commits. Then try merging the branch into main. Does it work?

Properly done, at the end of these steps, the main branch will hold the complete code for the project. The HTML and CSS branches will only hold parts.

Bonus: Try merging main into the css-styles branch. Is this allowed?

Tip

Remember that you can view your webpage by opening the local HTML file in your browser.

18.8.1.2. Merge Conflicts¶

The best way to learn how to deal with merge conflicts is to solve merge conflicts! Avoiding them is preferred, but some will sneak in anyway.

Right now, the main and css-styles branches have the same code in the style.css file. Time to bring in some conflicts…

  1. Return to main. Open the CSS file and make the following changes:
    1. In one of your existing element selectors, change the color property for the text.
    2. Add another class selector and assign it some property values.
    3. Add an id selector and assign it some property values.
  2. Save and commit your changes.
  3. Move into the css-styles branch and make the following changes:
    1. In the SAME element selector you changed in main, choose a different color for the text.
    2. Add another class selector. Give it the same name as in main. For the properties, keep the first one the same but change the other(s).
    3. Add an id selector and assign it some property values. Give this selector a different name than the one defined in main.
  4. Save and commit your changes.
  5. Return to main and try merging in the css-styles branch. Spoiler alert! There will be some merge conflicts.

As you resolve the conflicts, consider the following:

  1. Which changes from parts a - c resulted in conflicts?
  2. For which conflict would the option to Accept Both Changes be a good choice?
  3. When does changing the same file in two different branches result in a conflict?

Don’t forget to commit again!

18.8.2. Set up a Github Account¶

The chapter project and Assignment #5 both involve using GitHub.

Before you attempt these two tasks, check with your teacher for permission to create a free account. Depending on your school and your course, you might use a different service to create and share remote repositories.

If you will be using GitHub, create your account by following these steps:

  1. Navigate to GitHub’s site using the link above.
  2. Sign up for an account on the homepage either by filling out the form or clicking the Sign Up button.
  3. Once you have an account, you are ready to store your remote work!
